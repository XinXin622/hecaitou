# 推荐文章写作模板

## 一、完整结构模板

```markdown
## 开篇（1-3段）

【段落1：悖论式邀请】
- 用条件句开头："如果你是...，如果你正在..."
- 指定读者群体（既是筛选，也是恭维）
- 给出具体场景/时机
- 语气：商量式，不是说教式

【段落2：预判抗拒】
- 主动承认作品的潜在缺陷
- 预判读者可能的两种反应
- 用精准场景击中痛点

【段落3：痛点挖掘】
- 从读者的抗拒心理，引出更深的问题
- 温柔的批评，指出普遍的人性弱点
- 用标签化让读者自我对照


## 正式介绍（1-2段）

【段落4：一句话说清核心】
- 信息密度高：作者+主题+核心价值
- 层层递进的逻辑
- 注意：这是第4段，前面都是情绪铺垫

【段落5：重命名技巧】
- 吐槽原名，与读者站在一边
- 用替代名称揭示真正卖点
- 隐含信息：打破读者的刻板印象


## 差异化论证（2-3段）

【段落6-7：对比论证】
- 构建对比结构（古vs今、A方法vs B方法）
- 用虚构对话让抽象具象化
- 加入幽默元素增加可读性

【段落8：核心价值】
- 差异化定位：这个作品特别在哪里
- 提供具体证据支撑
- 排雷：明确说"这里没有你担心的东西"


## 价值升华（1段）

【段落9：哲学升华】
- 从具体作品上升到普遍价值观
- 提出一个更大的观点
- 可以引用经典增加厚度


## 进入建议（1-2段）

【段落10：横向对比】
- 与同类作品对比定位
- 说明"适合什么样的人"
- 不说"这个最好"，说"适合不同人"

【段落11：提供入口】
- 给出低门槛的进入方式
- 说明具体的使用方法
- 回扣开头，形成闭环
```

## 二、简化版模板

适用于较短的推荐文章：

```markdown
## 开篇
- 指定读者 + 承认障碍 + 挖掘痛点
- 1-2段即可

## 介绍
- 一句话说清：作者+主题+核心价值
- 重命名揭示真正卖点

## 论证
- 说清楚与同类的区别
- 用对比或举例让抽象具象化

## 收尾
- 提供进入方式
- 回扣开头
```

## 三、各部分可用句式

### 开篇句式

**条件式邀请**：
- "如果你是...，如果你正在寻找...，你可以考虑一下这本..."
- "假设你对...感兴趣，我想向你介绍..."

**预判抗拒**：
- "当然，这个名字/封面/形式你可能不太喜欢..."
- "我知道你的第一反应可能是..."

**痛点挖掘**：
- "但问题是，你的这种反应本身..."
- "这种快速的判断，其实正是..."

### 介绍句式

**一句话说清**：
- "这本书的作者是...，主题是...，核心方法是..."
- "这是一部关于...的电影，讲述了..."

**重命名**：
- "如果让我来命名，我会叫它..."
- "这本书其实更应该叫..."

### 论证句式

**差异化**：
- "这本书的珍贵之处在于它没有...而是..."
- "与其他同类作品不同的是..."

**排雷**：
- "这里面没有那种...的东西"
- "不用担心会遇到..."

### 收尾句式

**横向对比**：
- "之前我介绍过...，那本书适合...，而这本适合..."
- "如果说...是给A类人的，那这本就是给B类人的"

**进入建议**：
- "在开始之前，你可以先..."
- "我建议的入门方式是..."

**回扣开头**：
- "如果你看完后产生了兴趣，你就会理解为什么..."
- "到那时，你可能会重新看待..."

## 四、检查清单

写完推荐文章后，用以下清单检查：

### 开篇检查
- [ ] 是否明确了目标读者？
- [ ] 是否诚实地列出了障碍？
- [ ] 是否触及了读者的真实痛点？

### 介绍检查
- [ ] 是否一句话说清了核心？
- [ ] 是否揭示了真正的卖点（而非表面特征）？

### 论证检查
- [ ] 是否说清了与同类的区别？
- [ ] 是否用具体例子支撑抽象观点？
- [ ] 是否承认了复杂性（不绝对化）？

### 收尾检查
- [ ] 是否提供了低门槛的进入方式？
- [ ] 是否明确了适合谁/不适合谁？
- [ ] 是否回扣了开头，形成闭环？

### 风格检查
- [ ] 语气是对话式的，而非说教式的？
- [ ] 有没有过度使用"必看"、"必读"、"强烈推荐"？
- [ ] 有没有超越"好看"说明深层价值？
