# 写作练习工作流

用户指定一个原稿文件，按照以下流程完成分析和重写。

## 前置条件

- 写作风格提示词位于：`prompt/writing_style_prompt.md`
- 原稿文件由用户指定

## 工作流程

### 1. 读取文件

1. 读取用户指定的原稿文件
2. 读取 `prompt/writing_style_prompt.md` 写作风格提示词

### 2. 分析原稿问题

分析原稿在以下方面的问题：
- 开头方式（是否有具体场景切入）
- 信息密度与节奏（是否概念堆砌）
- 主观性（"我"的声音是否强烈）
- 过渡衔接（段落之间是否自然）
- 结尾力度
- 语言风格（是否口语化、有个人特色）

输出问题分析到同目录下：`{序号}_分析_{原稿标题}.md`

### 3. 按风格重写

根据 `writing_style_prompt.md` 的风格要求重写原稿，核心要点：
- 从个人经历切入开篇
- 强烈的第一人称视角
- 口语化书面语
- 连续反问、生动比喻
- 适度自嘲与幽默
- 敢于表达争议性观点
- 有力的结尾

输出重写稿到同目录下，让用户选择标题，命名为：`{序号}_重写_{标题}.md`

### 4. 重命名原稿

将原稿重命名为：`{序号}_原稿_{原标题}.md`

## 文件命名规则

使用两位数字序号前缀标识练习编号，例如：
- `01_原稿_xxx.md`
- `01_重写_xxx.md`
- `01_分析_xxx.md`

序号根据目录中已有的练习文件自动递增。

## 使用示例

用户输入：
```
/writing-practice @[练习/随笔01.md]
```

输出结果：
```
练习/
├── 02_原稿_随笔01.md
├── 02_重写_xxx.md
└── 02_分析_随笔01.md
```
